# Check Selection

Use the right measurement for the question.

## Bundlephobia

Use for published npm packages before adding a dependency. It reports minified and minified+gzip sizes, dependency count, dependency composition, export costs, history, and similar packages when the endpoints have data.

Good for:

- "How big is `react-router`?"
- "Compare these npm packages before I add one."
- "Scan this package.json for runtime dependency weight."

Limitations:

- It analyzes package-level imports, not the user's actual application bundle.
- It can fail for packages with missing dependency declarations or unsupported build setups.
- It does not measure private unpublished package code unless it is available on npm.

## npm pack / Publish Footprint

Use `npm pack --dry-run --json` for local package release footprint. This measures what npm would publish, including files that users download and install.

Good for:

- "Is this package shipping too many files?"
- "Are source maps/tests/docs included in the tarball?"
- "What is the package install footprint?"

## Local Build Artifacts

Measure `dist`, `build`, `lib`, `esm`, or explicit assets when the user cares about files already generated by the project.

Good for:

- "How large are our emitted JS/CSS files?"
- "Did this change grow the gzipped output?"
- "Which file is the largest in dist?"

## Actual App Bundle Analysis

Use the project's bundler stats and visualizers for app-level answers. Prefer the existing project tooling. If none exists, recommend adding a temporary analyzer:

- Webpack: `webpack-bundle-analyzer`
- Rollup/Vite: `rollup-plugin-visualizer`
- Next.js: `@next/bundle-analyzer`
- Source maps: `source-map-explorer`

Good for:

- "What code is actually in my production app bundle?"
- "Why did the route chunk grow?"
- "Which import pulled in this dependency?"

## Browser-Based Alternatives

Use browser/web checks when the task calls for them and network/browser access is available:

- `https://bundlejs.com/` for quick import-snippet experiments.
- `https://bundleinfo.com/` as a Bundlephobia-style package-size cross-check.
- `https://packagephobia.com/` for install-size style checks when accessible.

Treat these as supporting evidence, not a replacement for local project measurements.
